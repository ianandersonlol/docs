## Usage

```bash
sbatch run_boltz.sh input.yaml [additional_options]
```

### SLURM Configuration


## Input Format

Boltz2 uses YAML files to specify molecular complexes:

```yaml
version: 1
sequences:
  - protein:
      id: A
      sequence: MVLSPADKTNVKAAWGKVGAHAGEYGAEALERMFLSFPTTKTYFPHFDLSH
  - ligand:
      id: B
      smiles: 'CCO'
```

### Multi-chain Complex Example

```yaml
version: 1
sequences:
  - protein:
      id: A
      sequence: MVLSPADKTNVKAAWGKVGAHAGEYGAEALERMFLSFPTTKTYFPHFDLSH
  - protein:
      id: B
      sequence: MNIFEMLRIDEGLRLKIYKDTEGYYTIG
  - ligand:
      id: C
      smiles: 'CC(=O)OC1=CC=CC=C1C(=O)O'
```

### Supported Molecule Types

| Type | YAML Key | Value Format |
|------|----------|--------------|
| Protein | `protein` | Amino acid sequence |
| DNA | `dna` | DNA nucleotide sequence |
| RNA | `rna` | RNA nucleotide sequence |
| Ligand | `ligand` | SMILES string |

## Default Options

The runner script enables these options by default:
- `--use_msa_server`: Use online MSA server
- `--use_potentials`: Apply structure potentials

## Converting from Chai Format

If you have sequences in Chai Discovery FASTA format, use the converter:

```bash
python chai_to_boltz.py input.fasta output.yaml
```

### Chai FASTA Input

```
>protein|chain_A
MVLSPADKTNVKAAWGKVGAHAGEYGAEALERMFLSFPTTKTYFPHFDLSH
>ligand|small_mol
CCO
```

### Generated YAML Output

```yaml
version: 1

      id: A

      id: B

```

## Output

Boltz2 outputs predicted structures to the current directory or a specified output location.

## Comparison with Other Tools

| Feature | Boltz2 | Chai | AlphaFold 3 |
|---------|--------|------|-------------|
| Input format | YAML | FASTA | JSON |
| MSA server | Yes | Yes | Local DB |
| Memory requirement | 64 GB | 128 GB | 64 GB |
| Prediction time | ~12h | ~48h | ~24h |

<Tip>
  **Quick predictions:** Boltz2 is often faster than Chai for protein-ligand complexes. Consider using it for rapid screening.
</Tip>
