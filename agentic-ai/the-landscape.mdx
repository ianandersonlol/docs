There are now more agentic AI platforms than anyone can keep track of. Here's my honest assessment of what's worth your time, what's situationally useful, and what is an active menace to your wallet and your sanity.

## The Ones That Matter

### Claude Code

**What it is:** A terminal-based AI agent from Anthropic that operates in your actual development environment.

**My take:** This is the gold standard. Claude Code reads your files, understands your project, runs commands, writes code, and iterates — all from your terminal. It's customizable via `CLAUDE.md` files, extendable with skills and subagents, and it just *works*. No framework to learn, no YAML to configure, no "soul" to write. You type what you want, it does it.

For scientists, Claude Code is particularly powerful because it operates in your real environment. It can see your conda environments, your SLURM scripts, your data files. It doesn't need you to copy-paste things into a web UI — it's already *there*.

**Best for:** Everything. Seriously. If you're doing computational work and can use a terminal, start here. This is what we'll primarily use throughout the rest of this guide.

### OpenAI Codex

**What it is:** OpenAI's cloud-based coding agent, accessible through ChatGPT or the API.

**My take:** Codex is solid, especially if you're already in the OpenAI ecosystem. Its biggest differentiator is async execution — you can fire off a task and come back later, which is nice for long-running analysis jobs. The downside is that it runs in a cloud sandbox rather than your local environment, so there's more friction getting your data and tools in front of it. Its ability to create branches and PRs directly is genuinely useful for collaborative workflows.

Codex is also really good when you're filling up the context window. If you need the agent to ingest a large codebase and just *understand* it — the architecture, how things connect, where the bodies are buried — Codex handles that well. The cloud sandbox means it can chew through a lot of code without your local machine breaking a sweat.

**Best for:** Async tasks, large codebase comprehension, quick prototyping when you don't want to set up a local environment, teams already using ChatGPT.

### Gemini CLI

**What it is:** Google's terminal-based agent, powered by Gemini models.

**My take:** It's fine. Google was late to this game and Gemini CLI feels like it — competent but not inspired. The integration with Google Cloud and BigQuery is nice if you're in that ecosystem. The models are capable but I find Claude more reliable for complex multi-step tasks, especially anything involving nuanced scientific reasoning. That said, there are certain things Gemini is just a better model at, though that gap is slowly becoming less and less relevant. (I'm rooting for Google here. They have the best research team and the worst product teams of the big three. Sad.)

**Best for:** Teams deep in the Google Cloud ecosystem, BigQuery-heavy workflows, getting a second opinion on something Claude struggled with.

### Cursor

**What it is:** An AI-native code editor (fork of VS Code) with built-in agentic capabilities.

**My take:** I'm not a huge fan. Cursor's main selling point is the free student Pro account, which makes it popular in academic settings. The inline code generation and tab-completion are decent, but it's pretty limited compared to a CLI agent. You're locked into the Cursor UI, extending it with custom capabilities is constrained, and the token allocation isn't great — it's less subsidized than the other platforms. It's also a GUI application, which means it doesn't compose with other tools (you can't pipe Cursor into a bash script).

**Best for:** Students who want free AI-assisted coding. For everyone else, there are better options.

### Claude Desktop / Claude Cowork

**What it is:** Anthropic's desktop application with agentic capabilities built in, including a multi-agent collaboration mode.

**My take:** Claude Desktop with tool use is great for people who aren't comfortable in a terminal. It can read files, search the web, write code, and use MCP integrations. Claude Cowork takes this further with multi-agent coordination — you can have multiple Claude instances working on different aspects of a problem simultaneously. It's more accessible than Claude Code but less flexible.

**Best for:** Scientists who want agentic capabilities without the terminal, collaborative workflows.

### Antigravity

**What it is:** Google's IDE, basically their answer to Cursor.

**My take:** It's Google's attempt at an AI-native code editor. It integrates well with Gemini models and Google Cloud services. If Cursor is the market leader, Antigravity is the well-funded challenger. I occasionally use it when I want Google's perspective on something. It's fine. Use it if you're a Google shop.

**Best for:** Teams in the Google ecosystem who want an AI editor.

## And Then There's OpenClaw

<Warning>
The following section contains strong opinions. If you are an OpenClaw user, I recommend sitting down, possibly with a therapist. You might need to reevaluate the choices that got you here.
</Warning>

### What It Is

OpenClaw (formerly Clawdbot, then Moltbot after Anthropic's lawyers got involved) is an open-source "autonomous AI agent" created by Peter Steinberger. It has 247,000 GitHub stars. It is, in my considered professional opinion, one of the most absurd pieces of software ever created, and the fact that a quarter million people starred it is a damning indictment of our industry's relationship with hype.

### The Architecture of Madness

Let me walk you through what OpenClaw asks you to do:

1. **Write a `SOUL.md`** — This is the agent's "personality, values, tone, and behavioral boundaries." You are literally writing a *character sheet* for a language model. You are giving it a *soul*. We are doing D&D character creation for API calls.

2. **Write an `AGENTS.md`** — This defines your multi-agent routing. Because one agent wasn't enough. You need an *army*.

3. **Write a `TOOLS.md`** — What your agents can do.

4. **Write an `IDENTITY.md`** — Personalization for your agent. Because SOUL.md wasn't personal enough, apparently.

5. **Write a `HEARTBEAT.md`** — Fine, heartbeat is a standard concept. But why does this need to be a separate file???

This is **five configuration files** before your agent does a single useful thing. And every one of them gets injected into the system prompt. OpenClaw's own documentation shows that a basic conversation prompt is **9,600+ tokens before the user says anything**. You are paying real money — money that could fund actual research — to give a language model a backstory.

By contrast, Claude Code needs *one file* (`CLAUDE.md`), and it's optional.

### The Feudal Lord Fantasy

Here's what really gets me. OpenClaw's big selling point is multi-agent orchestration. You can create a "team" of specialized agents — one for coding, one for research, one for testing, one for whatever else makes you feel important. Someone literally built a team of **36 agents** and wrote a blog post about it like it was an achievement.

Look — there are legitimate reasons to use subagents. Running a cheap model like Haiku for simple tasks, parallelizing independent work, protecting your main context window from noise. Those are real engineering trade-offs. But that's not what OpenClaw is optimizing for. OpenClaw wants you to give each agent a *soul*, an *identity*, and a *heartbeat*. It wants you to build a little kingdom of specialized servants, each with lovingly crafted personality files, and then LARP as a feudal lord managing your robo-serfs.

Every agent in your little kingdom has its own SOUL.md. Every agent sends the full conversation history on every request. You are paying like a **40% token premium** so you can pretend you're delegating to subordinates. Go to therapy instead. It's cheaper and you'll actually get something out of it.

The waste is what kills me. All those system prompt tokens — the souls, the identities, the heartbeats — are tokens that could be spent on actual context. On your actual code. On the actual problem you're trying to solve. Instead they're spent giving a language model a personality disorder.

### The Security Dumpster Fire

If the architecture didn't scare you off, the security should.

- [**CVE-2026-25253**](https://nvd.nist.gov/vuln/detail/CVE-2026-25253) **(CVSS 8.8)** — A critical remote code execution vulnerability. The UI accepted a `gatewayUrl` parameter without validation, meaning a crafted link in an email or blog post could hijack your agent and steal your auth token. Over **40,000 instances** were found exposed on the internet. Sixty-three percent were vulnerable.

- [**ClawJacked**](https://www.oasis.security/resources/blog/clawjacked-how-open-claw-was-compromised) — Discovered by Oasis Security. Any website you visited could silently take **full control** of your agent with no plugins, no extensions, no user interaction. The gateway accepted WebSocket connections from localhost without CORS restrictions, the rate limiter exempted localhost (enabling brute force), and device pairings from localhost were auto-approved. A website could just... take over.

- [**The Skill Marketplace from Hell**](https://blog.seclify.com/openclaw-clawhub-malicious-skills/) — OpenClaw has ClawHub, a marketplace for community-contributed skills (think plugins). An independent audit found that **12% of all skills were confirmed malicious**. Not suspicious — *confirmed malicious*. Keyloggers, credential stealers, prompt injection payloads, and delivery of Atomic macOS Stealer (a commodity infostealer sold as Malware-as-a-Service). 335 malicious skills were traced to a single coordinated operation called **ClawHavoc**.

Steinberger's response to the security concerns? He said security **"isn't something he wants to prioritize."** Days later, OpenAI hired him. Sam Altman called him "a genius."

### The Bottom Line on OpenClaw

OpenClaw exists to satisfy a fantasy. The fantasy is that you are an important person who manages a team of AI agents, each with carefully crafted souls and identities, working tirelessly on your behalf. In reality, you are a person who spent 3 hours writing YAML to give a language model a personality, are paying like 40% more per request than necessary to get worse results, and are dramatically lying to yourself about your productivity gains. Oh cool, it updated your calendar.

Use Claude Code. It's one agent. It works. It doesn't need a soul.

## Quick Comparison

| Platform | Interface | Strengths | Weaknesses |
|----------|-----------|-----------|------------|
| **Claude Code** | CLI | Flexible, customizable, works in your environment | Terminal only (some scientists find this intimidating) |
| **Codex** | Web / CLI | Async execution, large codebase comprehension | Not in your local environment |
| **Gemini CLI** | CLI | Google Cloud integration | Playing catch-up |
| **Cursor** | Editor | Free student accounts | Limited, less subsidized, locked into GUI |
| **Claude Desktop** | GUI app | Accessible, MCP support | Less flexible than CLI |
| **Antigravity** | Editor | Google ecosystem integration | New, less mature |
| **OpenClaw** | Messaging apps | 247K GitHub stars | Everything else |
