Spreadsheets are where a lot of real work lives: forecasts, budgets, operating dashboards, customer exports, trial balances, inventory plans, and board reporting. Claude can help with all of that, but the best surface depends on what you are trying to change.

The simple rule:

<Card title="If the workbook itself is the work, start with Claude for Excel." icon="file-excel">
  Use the Excel add-in when you want Claude to inspect cells, trace formulas, update assumptions, preserve workbook structure, cite source cells, debug errors, or make changes directly inside Excel. Use chat, Cowork, and Claude Code around that core workflow when the job moves beyond one open workbook.
</Card>

## Before you start

Claude for Excel is an add-in you install inside Excel, not a separate app. Three things to check before your first session (all current as of July 2026 — see [Anthropic's setup guide](https://support.claude.com/en/articles/12650343-use-claude-for-excel) for the latest):

1. **You need a paid Claude plan.** Claude for Excel is available on Pro, Max, Team, and Enterprise plans. A free account will not work.
2. **Your Excel needs to be recent.** It runs in Excel on the web, Excel on Windows and Mac with a Microsoft 365 subscription, and Excel on iPad. Perpetual-license versions like Excel 2016 and 2019 are not supported, and neither is Excel on Android.
3. **On a work computer, IT may need to approve it.** Individuals install the add-in from the Microsoft Marketplace and sign in with their Claude account. In many companies, add-ins are locked down and an administrator has to deploy it through the Microsoft 365 admin center. If the add-in will not install, that is the first thing to ask about.

## The four surfaces

<CardGroup cols={2}>
  <Card title="Claude for Excel" icon="file-excel">
    Live workbook work: formulas, assumptions, scenarios, formatting, pivots, charts, model review, and cell-level citations. You still review every change — it is not a substitute for audit judgment.
  </Card>
  <Card title="Claude chat" icon="comments">
    Claude at claude.ai or in the desktop app. Best for thinking, explaining, drafting, and one-off files when the spreadsheet is not a long-lived model. Uploaded copies drift from the real workbook.
  </Card>
  <Card title="Claude Cowork" icon="folder-open">
    Longer knowledge-work tasks in the desktop app that touch files, folders, recurring work, or multiple outputs like a workbook plus a memo or deck. More autonomy means more scope — mind sensitive files.
  </Card>
  <Card title="Claude Code" icon="terminal">
    Project-style spreadsheet systems: folders, repeatable refreshes, scripts, validation checks, generated workbooks, and versioned workflows. Usually not the fastest tool for one cell in an open workbook.
  </Card>
</CardGroup>

This is the spreadsheet version of [Choosing the Right Tool](/agentic-ai/when-to-use-what): start simple, pick the smallest surface that can safely do the job, and add more machinery only when the workflow needs it.

## When to use each one

| You want to... | Use | Why |
|---|---|---|
| Explain a confusing formula | **Claude for Excel** | It can cite the formula cell and trace the precedents. |
| Change one assumption and see the model impact | **Claude for Excel** | The workbook stays live, and dependencies are preserved. |
| Find hard-coded values that should be formulas | **Claude for Excel** | It can inspect formulas and constants inside the workbook. |
| Brainstorm the right model structure before building | **Claude chat** | You are thinking, not editing yet. Keep the workbook untouched. |
| Clean a simple CSV and make a quick chart | **Claude chat** or **Claude Code** | Chat is fine for small one-offs. Claude Code is better if you want saved files, scripts, and repeatable outputs. |
| Refresh a monthly reporting package | **Claude Code** or **Cowork** | The task probably includes inputs, outputs, backups, validation, and commentary beyond one workbook. |
| Turn an Excel model into a memo or deck | **Cowork** or Microsoft 365 cross-app work | The job spans Excel, Word, PowerPoint, and maybe email. |
| Build a repeatable workflow you run every month | **Claude Code** | Put the workbook in a project folder, add rules, write checks, and save the routine. |
| Have several related spreadsheet tasks run in parallel | **Cowork** | It can coordinate longer tasks and sub-workstreams. |

## What if I use Google Sheets?

Claude for Excel only works in Excel — there is no equivalent add-in for Google Sheets today. You still have two reliable paths:

- **For reading and analysis**, connect Claude chat to Google Drive with the [Google Workspace connector](https://support.claude.com/en/articles/10166901-use-google-workspace-connectors). Claude can search and read your Sheets, but the connector is read-only: it cannot edit cells or write back to the sheet.
- **For editing**, download the sheet as `.xlsx` and work on the file with Claude chat, Cowork, or Claude Code, then re-import or share the result. That costs you the live-workbook loop, so treat the download as a snapshot and keep track of which version is current.

Anthropic's spreadsheet tooling is evolving quickly, so if Sheets is your daily driver it is worth rechecking the [Help Center](https://support.claude.com) every few months.
