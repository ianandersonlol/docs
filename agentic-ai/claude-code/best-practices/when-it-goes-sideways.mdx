Something will eventually go wrong. Claude will misunderstand the task, choose the wrong file, produce a chart that looks strange, or start down a path you did not intend.

That does not mean you broke anything. It means you need to steer.

## The recovery recipe

<Steps>
  <Step title="Stop the current path">
    If Claude is asking permission for something that looks wrong, reject it. If it is still working, tell it:

    > Stop. Do not make any more changes yet.
  </Step>
  <Step title="Ask what changed">
    Get a short inventory before you decide what to undo:

    > What files did you create, edit, rename, move, or delete in this session?
  </Step>
  <Step title="Decide whether to undo or redirect">
    If the work is close, redirect it:

    > Keep the cleaned spreadsheet, but redo the chart by quarter instead of by month.

    If the work is wrong, use your undo path:

    > Rewind to before the last set of edits.
  </Step>
  <Step title="Restart smaller">
    Break the task into a smaller step:

    > First only inspect the file and tell me the columns. Do not clean anything yet.
  </Step>
</Steps>

## Common recovery moves

| Problem | What to say |
|---|---|
| Claude picked the wrong file | "Stop. Use `sales-2025.csv`, not `sales-old.csv`." |
| It overwrote something you wanted preserved | "Restore the original from the previous version, then save future output as a new file." |
| The result looks suspicious | "Show me exactly how you calculated that number." |
| The folder got messy | "List every file you created and propose a cleanup plan. Do not move anything yet." |
| The conversation is confused | "Summarize the current state, then I will start a fresh session." |

## When to start over

Start a fresh session when:

- The chat is long and Claude seems forgetful.
- You changed your mind about the goal.
- There were several false starts.
- You cannot explain what state the workspace is in.

Fresh sessions are cheap. The files remain in the folder; you are just giving Claude a cleaner conversation.

## The calm reset prompt

If you only remember one recovery prompt, use this:

> Pause. Tell me what you changed, what files now exist, what assumptions you made, and the safest way to get back to the last good state. Do not change anything until I approve.

That prompt turns confusion back into a plan.
