Once the [core loop](/agentic-ai/claude-code/first-session/core-loop) feels natural and you've checked a few results with confidence, a whole second layer of Claude Code opens up. You don't need any of this to be productive — but each one removes friction once the basics are second nature.

## 1. Teach Claude your preferences once

If you keep repeating the same instructions every session — *"work on copies," "show your assumptions," "keep summaries short"* — you can write them down **once** in a project memory file (conventionally named `CLAUDE.md`). Claude reads it automatically every time you open that folder, so your preferences just... stick.

You don't have to create it by hand. Ask:

> Create a CLAUDE.md for this folder that says: always work on copies, never overwrite originals, and keep summaries to five bullets or fewer.

From then on, you stop repeating yourself. It's the highest-leverage "next step" for most people.

## 2. Save reusable shortcuts (slash commands)

If there's a task you do over and over — a weekly report, a standard cleanup — you can save it as a **slash command** and trigger the whole routine by typing `/its-name`. It's a saved prompt you can fire anytime, instead of re-typing the instructions.

## 3. Let Claude split the work

For bigger tasks, Claude can approach the job in separate passes: one pass extracts facts, another checks assumptions, another reconciles the final table. You do not need to configure anything to benefit from this. Just ask for it:

> Handle this in separate passes: extract the facts, verify the numbers, then combine the result.

That is the beginner-friendly version of [subagents](/agentic-ai/claude-code/best-practices/subagents-and-plugins): not more machinery, just better task decomposition.

## 4. Reusable capabilities (skills and plugins)

**Skills** teach Claude specialized, repeatable know-how it can apply across projects — a step beyond a saved prompt. When you find yourself wishing Claude "just knew how to do X the way I like," that's a skill.

**Plugins** are bigger packages of extra capabilities. They are useful when plain files and prompts are not enough, but they are still a "later" move: learn the core loop first, then add tools when a repeated need appears.

## 5. External integrations

MCP and hooks are powerful, but they are not beginner topics. They connect Claude to outside systems or event-driven workflows. Come back to them once you are already comfortable with folders, permissions, verification, and recovery.
