You can drive Claude Code almost entirely by typing plain English. Still, five controls are worth recognizing on day one. They are the difference between "I hope this works" and "I know how to steer it."

## 1. Code and New Session

The **Code** tab is where Claude works with a folder on your computer. **New session** starts a fresh conversation for a fresh task.

![The Code tab and New session button in Claude Desktop](/images/claude-code/code-tab-new-session.png)

Use a new session when you change tasks. Your files stay in the folder; only the conversation resets.

## 2. Local and Project Folder

At the bottom of a Code session, Claude shows where it is working. **Local** means this is a local session on your computer. The folder button beside it is the project folder Claude can see.

![Claude Desktop prompt controls with local and project folder buttons](/images/claude-code/prompt-controls.png)

If Claude says it cannot find a file, check this strip first. You may simply be in the wrong folder.

## 3. The Prompt Box

This is where you describe the task:

> Look at the files in this folder and tell me what you see. Do not change anything yet.

That one sentence is a good first move in almost every new workspace. It confirms Claude can see the files and gives you a chance to catch a wrong folder before any work starts.

## 4. Accept Edits

When Claude changes files, the **Accept edits** control is your review gate. The exact UI can change over time, but the habit stays the same: look at what changed before you treat the task as finished.

You do not need to read code to use this. For beginner tasks, review the result file, the chart, the renamed folder, or the summary Claude produced.

## 5. Model, Effort, and Usage

The model and effort controls live near the lower-right corner of the session.

![Claude Desktop model and effort controls](/images/claude-code/model-effort-controls.png)

Beginners can leave these alone. The defaults are usually right. Later, you can use them to trade speed, usage, and reasoning depth on purpose. See [Choosing a Model & Effort](/agentic-ai/claude-code/best-practices/choosing-a-model).

## The tiny cheat sheet

| Control | What it means | Beginner habit |
|---|---|---|
| **Code** | Work with files in a folder | Use this guide here, not in normal chat |
| **New session** | Start a fresh conversation | One task per session |
| **Project folder** | The folder Claude is working in | Check it when files seem missing |
| **Accept edits** | Review changed files | Do not skip review |
| **Model / Effort / Usage** | Speed, depth, and plan usage | Leave alone until you have a reason |
