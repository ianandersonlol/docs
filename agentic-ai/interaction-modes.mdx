There are fundamentally four ways to interact with an AI agent. Each has trade-offs, but I have a strong preference — and after you try them all, I think you will too.

## The Four Modes

### 1. Command Line Interface (CLI)

**Examples:** Claude Code, Gemini CLI, Codex CLI

You open a terminal, type a command, and talk to the agent in a conversational loop. The agent can see your filesystem, run commands, and work directly in your environment.

```bash
# Start Claude Code in your project directory
cd ~/projects/my-protein-pipeline
claude
```

That's it. You're now talking to an agent that can see your entire project, run your scripts, and modify your files.

**Why this is my recommendation:**

- **Composability** — CLI tools play nicely with everything else. You can pipe output, use them in scripts, chain them with other tools. Try doing that with a GUI.
- **Environment access** — The agent sees your real conda environments, your real data, your real SLURM config. No uploading, no copy-pasting.
- **Flexibility** — You can customize behavior with config files, skills, and slash commands. You can run multiple instances. You can background tasks.
- **Speed** — No app to open, no UI to navigate. You're already in the terminal (you're a scientist, of course you are).
- **Reproducibility** — Everything you do is in text. You can save conversations, share commands, version-control your agent configuration.

**The downside:** If you've never used a terminal before, there's a learning curve. But if you're doing computational science, you already use the terminal. And if you don't, the [Mac Setup guide](/misc-bioinfo/mac-setup) will get you there.

### 2. Native Applications

**Examples:** Claude Desktop, ChatGPT

Dedicated apps with their own UI. You interact through a chat interface, sometimes with file upload, sometimes with tool integrations.

**The good:**
- Lower barrier to entry — if you can use iMessage, you can use Claude Desktop
- Visual features — image generation, file previews, rich formatting
- MCP integrations in Claude Desktop give it genuine agentic capabilities

**The less good:**
- You're copying and pasting between the app and your actual work environment
- Harder to automate or integrate into workflows
- The app controls the experience — you can't extend it the way you want

**When to use this:** If you're a wet lab scientist who wants to ask Claude to help interpret some data or write a script, Claude Desktop is perfectly fine. You don't need the CLI for everything. But for sustained computational work, you'll eventually want more flexibility.

### 3. Editor Plugins / AI Editors

**Examples:** Cursor, Antigravity, GitHub Copilot, VS Code + Continue

AI capabilities embedded directly in your code editor. The agent can see your open files, suggest completions, and make edits inline.

**The good:**
- Tight integration with your editing workflow
- See suggestions in context, accept/reject inline
- Tab completion is genuinely useful for boilerplate

**The less good:**
- You're limited to what the editor exposes — can't easily run arbitrary shell commands or complex multi-step workflows
- Editor plugins tend to be code-focused, not great for broader research tasks
- Cursor in particular locks you into their fork of VS Code

**When to use this:** If you're spending most of your time writing code in an editor (as opposed to running pipelines, analyzing data, etc.), an AI editor is a nice complement to a CLI agent.

### 4. Orchestration Layers

**Examples:** Claude Cowork

Meta-tools that manage multiple agents working in parallel.

**The good:**
- Run several tasks simultaneously — one agent on your docking pipeline, another on your analysis scripts
- Project-level overview of what all your agents are doing
- Shared context between workspaces

**The less good:**
- Additional complexity — now you're managing the manager
- Overkill for single-task workflows

**When to use this:** When you have genuinely parallel work. If you're setting up three different computational experiments, having them run simultaneously instead of sequentially is a real time saver.

## My Setup

Here's what I actually use day-to-day:

1. **Claude Code** as the primary workhorse — the vast majority of my agent interactions happen here
2. **Codex CLI** for certain tasks that involve holding a lot of information in the context window, or when I want to compare approaches with Claude Code. The desktop app is also nice for managing multiple parallel agent threads visually
3. **Antigravity / Gemini CLI** occasionally, when I want Google's opinion on something. There are certain things Gemini is just a better model at, though that gap is slowly becoming less relevant
4. **Claude Desktop** for quick questions when I'm away from the terminal

Your mileage will vary. The beauty of the CLI-first approach is that it doesn't lock you out of anything else — Claude Code works great alongside any editor, any app, any other tool.
