# Customizing Your Shell: The Power of .zshrc

Now that you have all the tools installed, we can customize your shell to make your workflow better and more efficient. The `.zshrc` file is your shell's configuration file that runs every time you open a new terminal session. Think of this as your settings file. `rc` stands for "run commands" which is essentially the commands that get run when zsh (Apple's version of bash) runs at the start!

## Basic .zshrc Setup

```bash
# Open your .zshrc file
code ~/.zshrc
```

Add these useful configurations:

## Useful Functions

```bash
# ============================================
# USEFUL FUNCTIONS
# ============================================

# Quick sequence statistics
seqstats() {
    if [ -z "$1" ]; then
        echo "Usage: seqstats <fasta_file>"
        return 1
    fi
    echo "File: $1"
    echo "Number of sequences: $(grep -c '^>' $1)"
    echo "Total bases: $(grep -v '^>' $1 | wc -m)"
    echo "File size: $(du -h $1 | cut -f1)"
}

# Create and enter directory
mkcd() {
    mkdir -p "$1" && cd "$1"
}

# Extract various archive formats
extract() {
    if [ -f $1 ]; then
        case $1 in
            *.tar.bz2)   tar xjf $1     ;;
            *.tar.gz)    tar xzf $1     ;;
            *.bz2)       bunzip2 $1     ;;
            *.rar)       unrar x $1     ;;
            *.gz)        gunzip $1      ;;
            *.tar)       tar xf $1      ;;
            *.tbz2)      tar xjf $1     ;;
            *.tgz)       tar xzf $1     ;;
            *.zip)       unzip $1       ;;
            *.Z)         uncompress $1  ;;
            *.7z)        7z x $1        ;;
            *)           echo "'$1' cannot be extracted via extract()" ;;
        esac
    else
        echo "'$1' is not a valid file"
    fi
}

# Quick BLAST database search
blastdb() {
    if [ -z "$1" ] || [ -z "$2" ]; then
        echo "Usage: blastdb <query.fasta> <database>"
        echo "Example: blastdb protein.fasta nr"
        return 1
    fi
    blastp -query $1 -db $2 -outfmt 6 -max_target_seqs 10
}
```

## Prompt Customization

Your prompt is what you see before each command in the terminal. Customizing it can provide useful information at a glance, like your current directory or active conda environment.

```bash
# ============================================
# PROMPT CUSTOMIZATION
# ============================================

setopt PROMPT_SUBST
PROMPT='%F{green}%n@%m%f:%F{blue}%~%f${CONDA_DEFAULT_ENV:+ (%F{yellow}$CONDA_DEFAULT_ENV%f)}$ '
```

### Breaking Down the Prompt

Let's understand what each part does:

| Component | Meaning |
|-----------|---------|
| `setopt PROMPT_SUBST` | Enables variable substitution in the prompt (required for dynamic content) |
| `%F{green}` | Start coloring text green |
| `%n` | Your username |
| `@` | Literal @ symbol |
| `%m` | Your machine's hostname |
| `%f` | Reset color to default |
| `:` | Literal colon separator |
| `%F{blue}` | Start coloring text blue |
| `%~` | Current directory (with `~` for home) |
| `%f` | Reset color to default |
| `${CONDA_DEFAULT_ENV:+ (...)}` | If a conda environment is active, show it in parentheses |
| `%F{yellow}$CONDA_DEFAULT_ENV%f` | The environment name in yellow |
| `$ ` | The actual prompt character followed by a space |

### Example Output

With this prompt, your terminal will look like:

```
yourname@MacBook:~/projects/analysis (genomics)$
```

Where:
- `yourname@MacBook` is green (your user and machine)
- `~/projects/analysis` is blue (current directory)
- `(genomics)` is yellow (active conda environment, only shown if one is active)

### Other Useful Prompt Variables

| Variable | Description |
|----------|-------------|
| `%d` | Full path to current directory |
| `%T` | Current time (24-hour HH:MM) |
| `%t` | Current time (12-hour with am/pm) |
| `%D` | Current date (YY-MM-DD) |
| `%?` | Exit status of last command |
| `%#` | Shows `#` if root, `%` otherwise |

## Reload Your Configuration

After editing your `.zshrc`, reload it:

```bash
source ~/.zshrc
```
